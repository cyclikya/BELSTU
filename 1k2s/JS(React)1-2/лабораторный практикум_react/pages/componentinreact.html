Компоненты

Компоненты позволяют разбить интерфейс на независимые части, про которые легко думать в отдельности. Их можно складывать вместе и использовать несколько раз.

Во многом компоненты ведут себя как обычные функции JavaScript. Они принимают произвольные входные данные (так называемые «пропсы») и возвращают React-элементы, описывающие, что мы хотим увидеть на экране.

Самый простой способ определить компонент — написать JavaScript-функцию:

class Welcome extends React.Component {
 render() {
   return <h1>Привет, {this.props.name}</h1>;
  }
}

Данная функция — корректный компонент React, потому что он принимает один аргумент-объект «props» (который обозначает свойства) с данными и возвращает элемент React. Такие компоненты мы называем «функциональными», потому что они являются буквально функциями JavaScript.

Начало работы

Для рендеринга элемента на веб-странице применяется метод ReactDOM.render(), который принимает два параметра:

1. Первый параметр представляет компонент, который мы хотим отобразить на веб-странице. В данном случае это обычный заголовок. В этот метод передается элемент, который надо добавить на веб-страницу.

2. Второй параметр - это тот элемент веб-страницы, в котором будет производиться рендеринг компонента из первого параметра. В данном случае это элемент <div id="app">.

Как отрендерить компонент

Раньше мы сталкивались только с элементами React, представляющие DOM-теги:

const element = <div />

Однако элементы также могут быть пользовательскими компонентами:

const element = < Welcome name = "Алиса" />

Например, этот компонент выведет «Привет, Алиса» на страницу:

function Welcome (props) {
  return <h1>Привет, {props.name}</h1>;
}

const root = ReactDOM.createRoot(document.getElementById('root'));
const element = <Welcome name="Алиса" />;
root.render(element);

Объявление функции Welcome, которая принимает props в качестве аргумента. Внутри функции определено JSX-выражение, которое отображает приветствие с использованием значения props.name.

Объявление константы element, которая содержит JSX-элемент, вызывающий компоненту Welcome и передающий ей атрибут name со значением "Sara".

Использование ReactDOM.render() для рендеринга JSX-элемента element на странице. Функция ReactDOM.render() принимает два аргумента: рендеряемый элемент (в данном случае element) и элемент DOM, в который будет вставлен результат рендеринга (в данном случае элемент с идентификатором 'root').

Состояния

Компоненты React могут иметь состояние, то есть объект, содержащий информацию о том, как этот компонент должен отображаться и вести себя.

В случае изменения состояния компонента, вызывается метод render, который перерисовывает компонент на экране. Так, можно сказать, компонент реагирует на изменение состояния.

Также можно оперировать состоянием приложения с помощью менеджера состояний, например Redux или ContextAPI

Композиция компонентов

Компоненты могут ссылаться на другие компоненты в своём выводе. Это позволяет использовать одну и ту же абстракцию компонента для любого уровня детализации. Кнопка, форма, диалоговое окно, экран: в приложениях React все они обычно являются компонентами.

Например, компонент App может отрендерить компоненты несколько раз:

function Welcome (props) {
  return <h1>Привет, {props.name}</h1>;
}

function App() {
  return (
  <div>
   <Welcome name="Алиса" />
   <Welcome name="Маша" />
   <Welcome name="Аня" />
  </div>
 ) ;
}

Таким образом:
Компоненты играют важную роль в React, обеспечивая модульность, переиспользование и управляемость кода, что делает разработку пользовательского интерфейса более эффективной и удобной.
